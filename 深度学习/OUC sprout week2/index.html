DataLoader里面一些参数解释

  • 对torchvision.datasets.MNIST(root, train=True, transform=None, target_transform=None, download=False)

    • root 为数据集下载到本地后的根目录，包括 training.pt 和 test.pt 文件

    • train，如果设置为True，从training.pt创建数据集，否则从test.pt创建。

    • download，如果设置为True, 从互联网下载数据并放到root文件夹下

    • transform, 一种函数或变换，输入PIL图片，返回变换之后的数据。

    • target_transform 一种函数或变换，输入目标，进行变换。

  • batch_size：每个batch的大小

  • shuffle：是否进行shuffle操作，即每个 epoch 开始时，数据是否被随机打乱操作。

  • num_workers：加载数据的时候使用几个子进程

创建全连接网络和CNN

分别在小型全连接网络和CNN上训练与测试，可以看到，CNN的效果要比全连接网络强

打乱像素顺序再次在两个网络上训练与测试，可以看出，此时全连接网络性能变化不大，但是CNN的性能有所下降，不如全连接网络，这是由于像素的打乱，使得CNN无法很好利用像素间的局部关系了

问题总结

dataloader ⾥⾯ shuffle 取不同值有什么区别？

**shuffle=True**：每个 epoch 开始时，数据会被随机打乱。这可以防止模型学习到数据顺序的偏差，尤其是在训练集中数据按类别或其他规律排列时。打乱数据有助于提高模型的泛化能力。

**shuffle=False**：每个 epoch 开始时，数据按原始顺序加载。通常用于测试集或验证集，因为这些数据只需要前向传播，不需要随机化。

transform ⾥，取了不同值，这个有什么区别？

Dataloader里的transform参数通常定义对数据的预处理操作。对于训练集，多使用RandomCrop（随机裁剪）、RandomHorizontalFlip（随机翻转）、ToTensor、Normalize（归一化）等操作用于增强数据多样性，防止过拟合。而对于测试集，一般只有基本的ToTensor、Normalize（归一化）操作，因为其用于评估模型性能，无需对原数据进行增强。

epoch 和 batch 的区别？

epoch 表示模型完整的遍历一次数据集的次数

batch 表示训练时一次输入模型的数据量（样本数量）

例如一个数据集有10000张图片，设置batch_size=100，则 10,000 张图片需要 100 次迭代完成 1 个 epoch

1x1的卷积和 FC 有什么区别？主要起什么作⽤？

1x1 卷积：

  • 可以对特征图进行通道维度（C）的变换（升维或降维）而不改变其空间尺寸（H、W），可处理任意尺寸的输入
  • 参数量小，减少了计算量
  • 局部关联，参数共享

全连接层（FC）：

  • 将特征图展平后映射到输出维度，会丢失空间信息
  • 参数量大，计算量大，可能导致过拟合

residual leanring 为什么能够提升准确率？ 1

  • 残差学习可应用于深层网络，用于链式求导特性，它能很好解决梯度消失的问题
  • 在残差结构中，在神经网络学习到这层的参数是冗余的时候它可以选择直接走这条“跳接”曲线，跳过这个冗余层，而不需要再去拟合参数使得输出H(x)等于x

代码练习⼆⾥，⽹络和1989年 Lecun 提出的 LeNet 有什么区别？

  • 激活函数：原版使用tanh函数或sigmoid函数作为激活函数，练习里使用的是 ReLU函数。
  • 池化层：原版使用平均池化，练习里使用的是最大池化。

代码练习⼆⾥，卷积以后feature map 尺⼨会变⼩，如何应⽤ Residual Learning?

原feature map可以通过卷积进行降采样去匹配输出feature map的尺寸，反之也可以考虑。

有什么⽅法可以进⼀步提升准确率？

  • 使用数据增强：旋转、翻转、裁剪、缩放等
  • 使用数据预处理：归一化、标准化、主成分分析（PCA）去相关等
  • 使用预训练模型在其他数据集上进行微调
  • 采用SGD + Momentum或Adam等优化器
  • 使用Dropout方法防止过拟合
