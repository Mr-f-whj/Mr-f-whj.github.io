学习内容

ResNet

提出背景

  • 网络加深可能导致梯度爆炸/消失问题

    过去方案：数据的标准化处理、权重初始化

    新想法：BN

  • 退化问题：层数深的网络没有层数浅的网络效果好。随着网络层数的增加，训练误差和测试误差反而会增大。这不是过拟合（因为训练误差也增大），而是表明深度网络难以优化，难以学习到恒等映射或简单的变换。

    新想法：使用残差结构

亮点

  • 超深的网络结构（突破1000层）
  • 提出residual（残差）结构
  • 使用Batch Normalization（批归一化）加速训练(丢弃dropout)

残差学习结构

定义：假设我们期望某几层堆叠的非线性层（不一定是整个网络）能够拟合一个复杂的底层映射 H(x)。Residual Learning 提出，让这些层（残差块里的主干）学习一个残差映射F(x)：F(x) = H(x) - x。那么最终原始映射H(x)就重构为：H(x) = F(x) + x。

基础的残差块结构如下：

  • 输入 x（特征矩阵）。

  • 经过两层（或更多层）带权重和非线性激活函数（如 ReLU）的变换，得到 $F(x,W_i)$。

  • 将原始输入 x 与变换后的输出 F(x) 相加（矩阵元素逐元素相加）：$H(x) = F(x,W_i) + x$。

    注意：F 的结果与 x的shape必须相同，不同时可使用1x1的卷积降维或升维。

  • 对相加后的结果应用激活函数：$ y = ReLU(H(x))$。

优势：

  • 当最优映射接近恒等映射 (H(x) ≈ x) 时，学习残差 F(x) ≈ 0 比学习完整的恒等映射 H(x) = x 要容易得多。

  • 快捷（跳跃）连接不会引入额外的参数，不会增加计算复杂度，而且即使网络层学到的 F(x) 效果不佳或者为零，输出 y 至少还能等于 x，保证了网络的性能不会比浅层网络更差，缓解了退化问题。

  • 缓解梯度消失/爆炸：。在反向传播时，梯度可以通过跳连路径直接、无损地传递到更浅的层。这使得深层网络的训练更加稳定和高效。

    d(loss)/dx = d(loss)/dy * (dF/dx + 1)。即使 dF/dx 很小（趋近0），+1 项也能保证梯度不会完全消失。

Batch Normalization（批归一化）

优质参考：Batch Normalization详解以及pytorch实验_pytorch batch normalization-CSDN博客

背景：对于Conv1来说输入的就是满足某一分布的特征矩阵，但对于Conv2而言输入的feature map就不一定满足某一分布规律了（注意这里所说满足某一分布规律并不是指某一个feature map的数据要满足分布规律，理论上是指整个训练样本集所对应feature map的数据要满足分布规律）。而我们Batch Normalization的目的就是使我们的feature map满足均值为0，方差为1的分布规律。

事实上，BN并不是对整个训练集的feature map的数据进行标准化处理，而是计算每一Batch的feature map进行标准化（batch越大越接近整个数据集的分布，效果越好）

具体步骤：

  • 对输入的每个Batch里的所有feature map， 计算每个通道维度上的均值 $μ_B$ 和方差 $σ_B²$。

    注意：均值 $μ_B$ 和方差 $σ_B²$是一个向量不是一个值，向量的第i个元素对应第i个通道维度的均值和方差。

  • 对该 Batch 中每个feature map的每个通道特征进行标准化：$x̂ = (x - μ_B) / √(σ_B² + ε)$(ε 是防止除零的小常数)

原理在上方说明。

BN、LN、IN 的主要区别：BN是对每一batch里不同样本的同一个通道channel进行标准化，LN是对每一batch里每一样本自身的所有通道channel进行标准化。IN是对每一batch里每一样本的每个通道channel上所有元素进行归一化。

为什么分组卷积可以提升准确率？既然分组卷积可以提升准确率，同时还能降低计算量，分组数量尽量多不行吗？

分组卷积可以提升准确率：分组卷积减少了运算量和参数量，对于相同输入输出大小的情况下，减少为原来的 1/g，g即分组数。

分组不能无限多：分组数量多虽然能提升准确率降低计算量，但是内存访问量会变大，过度的信息隔离和通道数的限制最终会导致模型学习到的特征不够丰富和有效，精度通常会显著下降。
