box-sizing：content-box | border-box | inherit 看 1

  1. content-box

    宽度和高度分别应用到元素的内容框，即在宽度和高度之外绘制元素的内边距和边框，最终元素的宽高要加上padding和margin。改变原宽高。

  2. border-box

    为元素设定的宽度和高度决定了元素的边框盒。就是说，为元素指定的任何内边距和边框都将在设定的宽度和高度内进行绘制。通过从已设定的宽度和高度分别减去边框和内边距才能得到内容的宽度和高度。改变原宽高。

  3. inherit

    规定应从父元素继承 box-sizing 属性的值。在这里我们使用的是content-box

- 函数原型

我用的是apply方法，这里memorize函数不是要返回一个新函数吗，谁调用了这个新函数，就把fn的this修改为这个新函数的调用者。函数的原型有bind方法，apply方法，还有call方法。它们的作用是让你可以明确指定函数执行时的 this 值，同时也允许你传入参数。

以下是它们之间的小差别：

  • bind：返回一个新函数，不会立即执行函数，可以指定 this 值，传递参数。所以我们可以定义一个变量接收一下修改this后的函数，有几个参数，就传几个。

  • call：立即执行函数，指定 this 值并传入参数列表，传参跟bind方法写法一致。

  • apply：同样立即执行函数，指定 this 值，同时需要将参数以数组形式传入。

不调用函数就单纯的想修改一下this，就使用bind，然后赋值给一个变量（后续使用）；想要修改this后立即执行就使用call或者apply。
