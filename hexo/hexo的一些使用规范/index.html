hexo博客写文章的一些基本规范

发表于2023-01-30|更新于2023-01-31|hexo
|阅读量:

1. 如何发布一个简单的文章

  • 首先打开本地博客的文件夹，右键用git bash打开，然后输入hexo new "文章的标题"便可在 source/_posts/ 里生成相应标题的.md文件，我们便可以使用Markdown编辑器在该文件中写文章了，而关于Markdown的一些基本语法我就不在这里一一赘述了，网上有许多介绍。

  • 撰写好文章后，我们便可以通过以下指令来完成我们的目的

  • 另外，你也可以在相应位置手动创建.md文件，注意手动在.md文件里加上front-matter，然后再进行生成和部署


2. 一些常见问题

  • 如何往Markdown文章里插入图片

    • 法一

    • 法二: 图片在文章和首页中同时显示，可以使用标签插件语法。语法：{% asset_img image.png 说明 %}

    • 法三：插入网络图片，语法：![图片标签](网络图片地址)


  • Hexo 设置显示文章摘要，首页不显示全文

    • 法一：Hexo 主页文章列表默认会显示文章全文，浏览时很不方便，可以在front-matter和正文中间插入 <!--more--> 进行分段。该代码前面的内容会作为摘要显示，而后面的内容（正文）会替换为 “Read More” 隐藏起来。

    • 法二：在front-matter添加填写参数description: （只在首页出现，正文不出现）


  • 讲一下front-matter的一些基本参数和生成.md文件时的配置
    在hexo生成文件时，会调用front-matter的参数来完成相关配置

    需要注意的点

    • tags和categories这两个参数比较重要
    • 一篇文章可以有单个或多个标签，多个标签时标签之间是同级
    • 一篇文章只允许有一个分类，不能同时分类到多个类别下，但是可以生成多级类别，上面的写法表示文章属于class1类里class2类的
    • 必须遵守严格的front-matter语法：key:空格+值，其他文件里的书写也是一样

  • 修改并部署后没有效果
    使用 hexo clean 清除缓存后重新部署。
