在开启关闭 Windows 功能界面中，开启 Hyper-V ，适用于 Linux 的 Windows 子系统，虚拟机平台，等待安装完成后重启电脑。

WSL 常用命令

命令 说明
wsl --list / -l 列出所有已安装的发行版
wsl --list --verbose / -l -v 显示详细信息（包括版本和默认发行版）
wsl --set-version <发行版名> 2 将指定发行版切换到 WSL2
wsl --set-default <发行版名> 设置默认运行的发行版
wsl --set-default-version 2 设置新安装的发行版默认使用 WSL2
wsl --shutdown 关闭所有运行中的 WSL 实例
wsl --terminate <发行版名> 强制终止指定发行版
wsl --unregister <发行版名> 注销并删除指定发行版
wsl --export <发行版名> <文件路径> 导出发行版到指定路径
wsl --import <发行版名> <路径> <文件> 导入发行版到指定位置

几点提示：

  • wsl1很老了，不建议用

  • wsl2要使用，微软要求 Windows 10 版本 2004 及更高版本（内部版本 19041 及更高版本）或 Windows 11 …….. 说是这么说，win10最好将系统更新到最新版本22H2，win11随意

    虽说20H以上也能用wsl2，因为博主之前尝试用docker desktop去部署navidrome，结果docker desktop下载总是启动失败（即使是参考对应版本下的），后来将win10更新到22H2问题迎刃而解，虽说之后放弃使用docker desktop（命令行响应太慢，而且时不时自动闪退），但以此提醒有其他想法的人。

参照网上即可。

几点提醒：

  • 使用命令行或是在微软商店下载可能会直接下载到C盘（不建议），有两种方式下载到其他盘
    1. （法1）先下载到C盘后将镜像打包移动到其他盘再启用，最后再删掉原来的发行版

win10

先启动wsl，在wsl的终端里输入hostname -I查看Ubuntu系统的IP地址，可能有多个，可以挨个试确认一下，即直接使用ubuntu IP地址:端口号去访问你创建的服务。

确认后根据该IP地址，在win的终端（管理员）里进行端口映射配置，配置完重启即可

listenport=外部宿主机端口，即你想要映射到外部的端口号
connectport=内部wsl虚拟机端口，即内部服务使用的端口号
connectaddress=执行hostname -I 查看的非127.0.0.1的Ubuntu的IP地址

上述命令表示，在宿主机上的1521端口，绑定访问wsl网络虚拟机172.17.160.27:4533的系统服务。
若同局域网用户想访问该wsl服务，正确访问地址应该是宿主机IP:4534，本机用户也可以直接localhost:4534访问。
